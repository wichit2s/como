ความซับซ้อน¶

อ้างอิงจาก 1 ตารางต่อไปนี้เปรียบเทียบขนาดข้อมูลและ algorithms ที่จะสามารถใช้ในการเขียนคำสั่งเพื่อให้ได้คะแนน

\(n\)

\(O\) ที่รับได้

ตัวอย่าง

\(\le 10\)

\(O(n!),O(n^6)\)

permutation ของ \(n\)

\(\le 15\)

\(O(2^n \times n^2)\)

TSP DP

\(\le 18\)

\(O(2^n \times n)\)

bitmask DP

\(\le 100\)

\(O(n^4)\)

3D DP + \(O(n), C(n,4)\)

\(\le 400\)

\(O(n^3)\)

\(\le 2K\)

\(O(n^2 log_2 n)\)

2-nested loops + tree-based DP

\(\le 10K\)

\(O(n^2)\)

2-nested loops, bubble/selection/insertion sort

\(\le 1M\)

\(O(n log_2 n)\)

merge sort, segment tree

\(\le 100M\)

\(O(n), O(log_2 n), O(1)\)
